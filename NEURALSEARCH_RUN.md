## 3. Запуск

Разовый запрос:

```bash
python -m app.NeuralSearch.main "что такое RAG и как он работает"
```




Пользователь вводит вопрос
LLM переформулирует запрос под веб-поиск
Поисковик ищет страницы в интернете
Страницы скачиваются и парсятся
Из каждой страницы достаются самые релевантные куски текста
Все найденные документы дополнительно переранжируются
LLM генерирует финальный ответ только по найденному контексту

То есть архитектурно это очень похоже на RAG over web:

Retrieve — найти документы
Rank — выбрать лучшие
Generate — сгенерировать ответ по контексту


Шаг 1
Пользователь пишет вопрос

Шаг 2
LLM переписывает вопрос в поисковую формулировку

Шаг 3
DuckDuckGo возвращает URL-кандидаты

Шаг 4
Каждый URL скачивается и очищается от HTML-мусора

Шаг 5
Текст страницы режется на чанки

Шаг 6
Из чанков выбираются самые релевантные к вопросу через TF-IDF + cosine

Шаг 7
Все документы из всех поисковых запросов объединяются

Шаг 8
Документы переранжируются через MMR

Шаг 9
Лучшие документы отправляются в LLM

Шаг 10
LLM генерирует финальный grounded-answer + литературу
